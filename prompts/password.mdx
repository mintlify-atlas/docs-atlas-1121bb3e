title: "Password"

---

## Installation

<CodeGroup>

```sh npm
npm install @inquirer/prompts
```

```sh yarn
yarn add @inquirer/prompts
```

</CodeGroup>

To install the standalone package instead:

<CodeGroup>

```sh npm
npm install @inquirer/password
```

```sh yarn
yarn add @inquirer/password
```

</CodeGroup>

## Usage

```js
import { password } from '@inquirer/prompts';
// Or
// import password from '@inquirer/password';

const secret = await password({ message: 'Enter your password', mask: true });
```

<Warning>
  The prompt does not store or log the entered value. However, if you pipe the result elsewhere in your application, take care not to expose it unintentionally.
</Warning>

## Options

<ParamField path="message" type="string" required>
  The question to ask the user.
</ParamField>

<ParamField path="mask" type="boolean">
  When `true`, each typed character is replaced with `*` in the terminal. When `false` (the default), input is invisible — nothing is shown as the user types.
</ParamField>

<ParamField path="validate" type="(value: string) => boolean | string | Promise<boolean | string>">
  Called on submit with the raw (unmasked) value. Return `true` to accept. Return a string to show it as an error message. Throwing a rejected promise is treated as a code error and will crash.
</ParamField>

<ParamField path="theme" type="Theme">
  Customize the look of the prompt. See [Theming](#theming).
</ParamField>

## Theming

Pass a partial `theme` object to override specific tokens. Unspecified keys fall back to the defaults.

```ts
type Theme = {
  prefix: string | { idle: string; done: string };
  spinner: {
    interval: number;
    frames: string[];
  };
  style: {
    answer: (text: string) => string;
    message: (text: string, status: 'idle' | 'done' | 'loading') => string;
    error: (text: string) => string;
    help: (text: string) => string;
  };
};
```
